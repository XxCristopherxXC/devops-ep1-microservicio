## Estrategia de Ramificación: GitFlow

Elegimos GitFlow porque nuestro proyecto tiene ciclos de desarrollo 
definidos y necesitamos separar el trabajo en progreso (develop) 
del código estable (main). Esto nos permite...

### Estructura de ramas:
- main: código en producción
- develop: integración de features
- feature/<nombre>: nuevas funcionalidades
- hotfix/<nombre>: correcciones urgentes en producción
